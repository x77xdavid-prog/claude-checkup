## 이 프로젝트의 규칙 (전역과 다른 것만)
- **샘플 프롬프트 필수** — `public/catalog.json` 의 모든 스킬은 `public/sample-prompts/{name}.json` 에 예시 프롬프트 파일을 **반드시** 가진다. 카탈로그에 스킬을 추가하면 같은 커밋에서 이 파일도 만든다.
  - 형식: `{ "name": "<카탈로그 name과 동일>", "prompts": ["...", ... 정확히 10개] }`
  - 프롬프트는 **한국어**, 사용자가 그 스킬에게 실제로 시킬 법한 자연스러운 요청. 10개는 서로 다른 실제 상황을 다룬다(쉬움→어려움, 단일→복합). 스킬 설명(description)에 근거해 그 스킬만의 기능에 특화.
  - 스킬명·영어 용어 나열 금지. 각 줄이 곧바로 붙여넣어 쓸 수 있는 완결된 지시문.
- API 응답은 `{ ok, data, error }` 봉투 형식.

## 작업 방식 (이 레포에서)
1. **재사용 먼저** — 새로 짜기 전에 이 레포·설치된 라이브러리·기존 패턴부터 확인 (ponytail 사다리).
2. **사다리** — 존재할 필요 있나? → 이미 있나? → 표준라이브러리? → 한 줄? → 그제서야 코드.
3. **증거 기반 완료** — 테스트/실행으로 확인하기 전엔 "됐다" 금지.
4. **로그 남기기** — 의미 있는 작업 후 `D:\프로젝트\_대시보드\logs\YYYY-MM-DD.md` 에 좋았던점/나빴던점 1줄씩.
